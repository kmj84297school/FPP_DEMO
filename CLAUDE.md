> 이 문서는 claude.ai 웹 세션에서 진행한 FPP 2학년 재구축 작업의 공식 인계본이다.

## 0. 프로젝트 소유자와 절대 원칙

- 소유자: 고명준 (세마고 2학년). 학생부종합전형 진로활동의 메인 프로젝트이자 상업화 검토 대상.
- **기록 정확성 원칙 (모든 산출물에 적용):**
  - 실제로 수행하지 않은 것을 완성된 것처럼 표현 금지
  - 성능 수치는 검증 방법과 함께 기록 (in-sample vs out-of-fold 구분 명시)
  - 한계는 숨기지 않고 문서화 — 이 프로젝트의 정체성은 "정직한 측정"
- 보고서·발표 문구 작성 시: "완성된 기술"과 "확장 계획"을 반드시 분리

## 1. 프로젝트 한 문단 요약

FBref 유래 공개 데이터(worldfootballR 아카이브, 빅5 리그 2018~2023)를 기반으로,
축구 선수의 현재 능력을 이중 가중(포지션 렌즈 + 스타일 렌즈)으로 점수화하고,
"2~3년 후 실제 성과"를 라벨로 XGBoost 2단 모델(빅5 잔존 분류 + 미래 능력 회귀)을
학습시켰으며, 선수 단위 클러스터 부트스트랩으로 예측 신뢰구간을 산출·검증했다.
1학년 프로토타입(실시간 크롤링 + 규칙 기반)의 한계를 해소한 재구축 버전이다.

```

```

## 4. 핵심 설계 결정 (변경 시 소유자 승인 필요)

1. **능력점수 = (포지션 렌즈 + 스타일 렌즈) / 2** — 포지션 라벨의 거칢을 스타일
   분류가 완충 (예: 공격형 풀백 TAA가 포지션 49.5/스타일 69.1로 구제됨)
2. **분위수는 시즌×포지션 풀(600분+) 내에서 직접 계산** — FBref 제공 퍼센타일 미사용
3. **스타일 판별은 구성비(shape) 피처로, 점수 가중과 지표 분리** — 순환성 방지
4. **라벨 = t+2~t+3 자격 시즌(900분+)의 능력점수 평균** — 단일 시즌 노이즈 축소
5. **이탈(빅5 소멸)은 별도 분류 모델** — 회귀에 섞지 않음 (생존편향 명시 처리)
6. **교차검증은 GroupKFold(선수 단위)** — 동일 선수 훈련/검증 동시 포함 = 누수
7. **부트스트랩은 선수 단위 클러스터 리샘플링** — 행 단위는 군집 구조 위반
8. **신뢰구간 = sqrt(sigma_bootstrap² + sigma_residual²) 결합** — 순수 부트스트랩은
   커버리지 29.3%로 실패 실측됨. 결합 보정 후 82.0%/52.1% (명목 80/50과 일치)

## 5. 검증된 수치 (인용 시 이대로)

| 항목 | 값 | 검증 방법 |
|---|---|---|
| 잔존 분류 AUC | 0.712 ± 0.026 | GroupKFold 5겹 |
| 미래 능력 회귀 MAE | 6.19 (R² 0.231) | GroupKFold 5겹 |
| 기저모델(미래=현재) 대비 개선 | +5.0% | 동일 폴드 정면 비교 |
| Δ(성장궤적) 피처 중요도 비중 | 25% | 폴드 평균 gain |
| 80% 구간 실측 커버리지 | 82.0% (보정 후) | 폴드 밖 표본 |
| 데모(2020 U21 상위 예측) | Mbappé, Vinicius, Foden, Haaland… | in-sample 주의 표기 |

- 코호트: 2018~2020 시즌, 만 23세 이하, 900분+, 1,323행(841명), 잔존율 75.1%
- 자체 발견: d_grp_progress(전진 능력의 성장분)가 Δ 피처 중 중요도 1위 —
  "수준보다 기울기가 미래 신호"

## 6. 남은 작업 (우선순위 순)

### A. 유사 선수 k-NN 레이어 (분석 기능 마지막 조각)
- 입력: 학습 행렬과 동일 피처 공간 (시즌 내 표준화 필수 — 시대 보정)
- 같은 포지션 그룹 내에서 (나이±1, 스탯 z-거리) k=10 최근접 과거 선수 검색
- 출력: 유사 선수들의 실제 t+2~t+3 결과 분포 → 실증 구간 + 사례 서술
- 주의: 유사도 거리가 멀면 "비교 신뢰도 낮음" 표시 (아웃라이어의 고독 문제)
- fbref_id로만 조인 (이름 조인 금지 — 동명이인 33명 존재 확인됨)

### B. FastAPI 웹 서빙 재구축 (핵심 작업)
- legacy의 app.py 구조 참고하되 신규 작성 권장
- 서버는 크롤링 없음: CSV + 모델 로드 → 선수 검색 → 즉시 응답
- 응답 구성: 현재 능력(이중 렌즈 분해) + 잔존 확률 + 예측 잠재력 + 80%/50% 구간
  + 유사 선수 비교 + 레이더/차트
- 구간 계산: mu ± z * sqrt(sigma_model² + 7.49²)  (z80=1.2816, z50=0.6745)
  sigma_model은 부트스트랩 앙상블 예측 표준편차 (앙상블 재학습 필요 시 boot_ci.py 참조)
- 주의: legacy app.py는 구식 TemplateResponse 규약 — 최신 starlette은
  `TemplateResponse(request, "x.html", ctx)` 순서
- 배포 목표: Render (기존 서비스 FPP_PROTOTYPE / fpp-prototype.onrender.com 계정 존재)
